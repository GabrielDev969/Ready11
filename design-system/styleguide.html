03 · O motor de bem-estar

Cores semânticas

Estas são as únicas cores saturadas do sistema. Cada uma mapeia um enum de backend e um significado no contexto do futebol. Nunca use estas cores decorativamente.

Wellness Status
EXCELLENT
Atleta em perfeitas condições de prontidão.
GOOD
Níveis normais de fadiga adaptativa.
WARNING
Sinal de atenção. Requer moderação de carga interna.
CRITICAL
Risco iminente de lesão ou overtraining severo.
Hydration Status
HYDRATED
Janela de hidratação segura para performance.
ATTENTION
Alerta de perda hídrica. Estimular protocolo de ingestão.
DEHYDRATED
Desidratação crítica. Risco de cãibras e queda de VO₂.

05 · Elementos

Iconografia

Ícones em linha, traço de 1.6px, grid 24×24, monocromáticos (herdam currentColor). Compatível com Lucide para produção. Biblioteca completa em icons.html.

06 · Elementos

Gráficos & visualização de dados

SVG leve, sem dependências. Linhas finas, sem grids pesados, cor do dado segue o status semântico. Galeria completa (linha dupla, heatmap, progresso) em dataviz.html.

09 · Implementação

Uso no Tailwind 3 + Django

Os tokens viram classes Tailwind. Exemplo de uma pill de status numa template Django:

<!-- template Django --> <span class="gss-pill gss-pill-critical"> <span class="gss-dot bg-critical-fg"></span> {{ athlete.wellness_status }} </span> <!-- ou com utilitários puros --> <div class="bg-elev border border-line rounded-md shadow-card p-5"> <p class="font-mono text-eyebrow uppercase tracking-widest text-fg3">Wellness médio</p> <p class="gss-metric-xl mt-2">4.2</p> </div>

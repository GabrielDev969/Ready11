{% trans "What's included" %}

{% trans "Schema isolation" %}

{% blocktrans %}Each customer (Workspace) lives in its own PostgreSQL schema via django-tenants. No data leaks between clients.{% endblocktrans %}

{% trans "Global identity" %}

{% blocktrans %}A single account works across workspaces, with email verification and a strict password policy out of the box.{% endblocktrans %}

{% trans "Role-based access" %}

{% blocktrans %}Per-workspace roles and permissions. The same person can be Owner in one workspace and Viewer in another.{% endblocktrans %}

{% trans "Invite-only onboarding" %}

{% blocktrans %}Controlled signup through expiring invitations, for both new company owners and internal team members.{% endblocktrans %}

{% trans "Hardened & i18n-ready" %}

{% blocktrans %}Security headers, brute-force protection and full internationalization — English and Brazilian Portuguese included.{% endblocktrans %}

{% trans "Docker-first" %}

{% blocktrans %}Ships with Docker, Gunicorn, WhiteNoise and a health check endpoint, ready to deploy behind a reverse proxy.{% endblocktrans %}

{% trans "How it works" %}

  1. 1

    {% blocktrans %}A super admin sends a "genesis" invitation to a future workspace owner.{% endblocktrans %}

  2. 2

    {% blocktrans %}The owner sets their name and company. The backend creates an isolated database schema on demand.{% endblocktrans %}

  3. 3

    {% blocktrans %}The owner invites their team and assigns roles. Members access the workspace on its own subdomain.{% endblocktrans %}

{% trans "Install & run" %}

{% blocktrans %}Requires Python 3.12, Docker and Node. The full guide lives in the README.{% endblocktrans %}

# 1. {% trans "Start the PostgreSQL database" %}
docker compose up -d

# 2. {% trans "Create the virtualenv and install dependencies" %}
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# 3. {% trans "Build the CSS (Tailwind)" %}
npm install
npx tailwindcss -i input.css -o static/css/output.css

# 4. {% trans "Apply migrations and create the public tenant" %}
python manage.py migrate_schemas --shared
python manage.py setup_public_tenant

# 5. {% trans "Run the development server" %}
python manage.py runserver

{% blocktrans %}Then open the app, create a super admin and send your first genesis invitation from the Django admin.{% endblocktrans %}

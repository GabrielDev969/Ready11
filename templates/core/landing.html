{% trans "Schema-isolated multi-tenancy" %}

{% blocktrans %}Each company gets its own PostgreSQL schema via django-tenants. Complete data isolation — no shared tables, no accidental leaks between clients.{% endblocktrans %}

acme.yourdomain.com → schema: acme

{% trans "Global accounts" %}

{% blocktrans %}One account, many workspaces. Email verification on signup. Strict password policy. Password reset by email. The same user can own one workspace and be a member of another.{% endblocktrans %}

{% trans "Roles & permissions (RBAC)" %}

{% blocktrans %}Per-workspace roles with granular permissions (users.invite, roles.edit, etc.). The Owner role is locked. All others are fully customisable by the workspace admin.{% endblocktrans %}

{% trans "Invite-only onboarding" %}

{% blocktrans %}Controlled access through expiring invite links (24h). Super admins send genesis invites to create new companies. Workspace owners invite their team. In-app notifications for existing users.{% endblocktrans %}

{% trans "Real-time notifications" %}

{% blocktrans %}WebSocket push via Django Channels + Redis. Personal, workspace-broadcast and global notifications. Unread badge in the topbar. 30-day retention with automatic cleanup.{% endblocktrans %}

{% trans "Audit log" %}

{% blocktrans %}Every important action is recorded: member invited/joined/removed, role created/updated/deleted, workspace settings changed. Stored in the public schema with 90-day retention.{% endblocktrans %}

{% trans "Security hardening" %}

{% blocktrans %}Brute-force protection (django-axes, 5 attempts → 1h lockout), HSTS, CSRF, secure cookies, content-type protection, clickjacking headers — all active in production by default.{% endblocktrans %}

{% trans "Observability" %}

{% blocktrans %}Request logging with method/path/status/duration. Optional Sentry for errors (HTTP + WebSocket). Structured JSON logs for CloudWatch, Grafana Loki or Datadog. Health check at /healthz/.{% endblocktrans %}

{% trans "i18n + user & workspace settings" %}

{% blocktrans %}English source strings with Brazilian Portuguese translation. Each user saves their preferred language — applies on every device on login. Each workspace sets its own timezone (default UTC).{% endblocktrans %}

{% trans "Flow" %}

{% trans "How it works" %}

1

{% trans "Genesis invite" %}

{% blocktrans %}A super admin creates an invite with no workspace assigned from the Django admin. The token link is printed to the server console.{% endblocktrans %}

2

{% trans "Workspace created" %}

{% blocktrans %}The owner opens the link, fills in name and company. The backend creates a dedicated PostgreSQL schema and subdomain on the fly.{% endblocktrans %}

3

{% trans "Team invited" %}

{% blocktrans %}The owner invites the team from /team/, assigns roles and configures permissions. Each member gets an email or an in-app notification.{% endblocktrans %}

4

{% trans "Ship your product" %}

{% blocktrans %}Multi-tenancy, auth, RBAC, notifications, audit and CI/CD are done. Add your own models inside the tenant schema and focus on the feature that sets you apart.{% endblocktrans %}

{% trans "Local setup" %}

{% trans "Running on your machine" %}

{% blocktrans %}You need Python 3.12, Docker Desktop and Node.js 18+. Everything else is installed by the commands below.{% endblocktrans %}

1

{% trans "Start the database and Redis" %}

docker compose up -d

{% trans "Starts PostgreSQL 17, Redis and Mailpit (email preview). Runs in background." %}

2

{% trans "Create the virtual environment and install dependencies" %}

python3 -m venv venv
source venv/bin/activate   # Windows: .\venv\Scripts\Activate.ps1
make setup                 # pip install + npm install + build CSS
3

{% trans "Configure environment variables" %}

cp .env.example .env
# Open .env — the defaults already work for local dev (DEBUG=True)
4

{% trans "Run migrations and create the public tenant" %}

make migrate    # migrate_schemas --shared + --tenant
make seed       # creates admin@example.com / Admin1234!
5

{% trans "Start the server" %}

make run    # http://127.0.0.1:8000

{% trans "Creating your first workspace" %}

  1. {% blocktrans %}Log in to /admin with admin@example.com{% endblocktrans %}
  2. {% blocktrans %}Under Workspace invites, create an invite with no workspace selected (genesis invite). The link appears in the server console.{% endblocktrans %}
  3. {% blocktrans %}Open the link, fill in your name and company — the backend creates an isolated PostgreSQL schema and a subdomain (company.localhost:8000) automatically.{% endblocktrans %}
  4. {% blocktrans %}Go to /team/ inside your workspace to invite the rest of the team.{% endblocktrans %}
